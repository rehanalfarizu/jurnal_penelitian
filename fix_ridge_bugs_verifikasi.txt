BUG #1: r2_buf_train selalu 0.000000 di setiap checkpoint
───────────────────────────────────────────────────────────────
Root cause: flush_retrain() memanggil del self._stream_buffer_X[:] SEBELUM 
_r2_buf_train dipanggil. Setelah buffer dihapus, self._stream_buffer_X 
berupa array kosong. Ketika _capture_checkpoint mengakses buffer, buf_X 
berupa np.array([]) — shape (0,), y_pred juga []. 

Hitungan R² buffer:
  ss_res = np.sum(([empty] - [empty])**2) = 0.0
  ss_tot = np.sum(([empty] - mean(empty))**2) = 0.0
  r2 = 1.0 - 0/0 → returns 0.0 (dari 'if ss_tot > 0 else 0.0')

Hasil: Semua checkpoint menunjukkan R² buffer = 0.000000 padahal model 
benar-benar fit pada data.

FIX: flush_retrain() sekarang RETURN (buf_X, buf_y) SEBELUM clear buffer.
Caller (maybe_retrain_chunk / streaming loop) bisa menghitung R² buffer 
dari tuple yang dikembalikan sebelum buffer dihapus.

Kode fix (Cell 2, flush_retrain):
  def flush_retrain(self):
      buf_X = np.array(self._stream_buffer_X)
      buf_y = np.array(self._stream_buffer_y)
      self._stream_buffer_X.clear()  # clear AFTER returning data
      self._stream_buffer_y.clear()
      self.update_model(buf_X, buf_y)
      return buf_X, buf_y  # ← FIXED: return BEFORE clear

VERIFIKASI:
  Buffer size before flush_retrain: 500
  flush_retrain returned: tuple
    buf_X shape: (500, 18)
    buf_y shape: (500,)
    Buffer size after flush_retrain: 0 (expect: 0)
  ✓ FIX #1 OK — data returned BEFORE buffers cleared
